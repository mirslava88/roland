# Presentation Display Manager — Проектный документ

## 1. Обзор проекта

**Название:** Presentation Display Manager (рабочее)  
**Цель:** Кросс-платформенное приложение (Windows / macOS) для бесшовного переключения между презентациями, видеороликами и PDF-документами на дополнительном экране в полноэкранном режиме.  
**Стиль:** Современный, тёмная тема.

---

## 2. Анализ существующих решений

### 2.1. Auto Presentation Switcher (APS) — PresentationTools

Наиболее близкий по концепции продукт. APS — коммерческое приложение, предназначенное для AV-техников на корпоративных мероприятиях.

**Что умеет:**
- Бесшовное переключение между PowerPoint, Keynote и PDF-презентациями по горячим клавишам и через Stream Deck / Bitfocus Companion.
- Показ скриншота последнего слайда поверх экрана во время загрузки следующей презентации — создаёт иллюзию бесшовного перехода.
- Медиаплеер (на базе LibVLC) для воспроизведения видео поверх экрана с fade-in/out.
- Показ статичных изображений и скриншотов поверх презентации (логотип, «заморозка» экрана).
- Интеграция со Stream Deck и Companion по сети, а также открытый TCP/IP API.
- Поддержка Windows 10/11 и macOS (Mojave — Sequoia).

**Чего НЕ делает (наши преимущества):**
- Нет предварительного просмотра слайдов в окне управления.
- Нет GUI-библиотеки презентаций — работа ведётся через файловую систему/папку.
- Нет встроенного управления видео (перемотка, таймлайн) — только play/pause/stop.
- Нет перехода к конкретному слайду по номеру.
- Нет единого интерфейса — управление разнесено между APS, Companion и внешними контроллерами.
- Платная подписка от $45/год за лицензию на 1 компьютер.

**Ключевой подход APS к PPTX:** APS не рендерит PPTX самостоятельно — он автоматизирует нативный PowerPoint (или Keynote) через COM-автоматизацию (Windows) и AppleScript (macOS). Это единственный надёжный способ сохранить все анимации, переходы и встроенное видео.

### 2.2. Другие рассмотренные решения

| Решение | Тип | Проблема для нашей задачи |
|---------|-----|--------------------------|
| **Visme, Canva, Beautiful.ai** | SaaS, облачные презентационные платформы | Создание, а не показ. Не работают с произвольными PPTX-файлами. |
| **Presenton** (open-source) | AI-генератор презентаций (Electron + FastAPI) | Генерация слайдов, а не управление показом. |
| **LibreOffice Impress** | Свободный офисный пакет | Частичная поддержка анимаций PPTX — многие эффекты теряются или отображаются некорректно. Не подходит для профессионального показа. |
| **PptxGenJS** | JS-библиотека | Создание PPTX, а не показ. Не рендерит существующие файлы. |
| **Google Slides** | Облачный сервис | Импорт теряет анимации и видео. Требует интернет. |

### 2.3. Выводы из анализа

1. **Полноценный рендеринг PPTX с анимациями и видео невозможен без нативного PowerPoint.** Ни одно стороннее решение (LibreOffice, браузерные конвертеры) не воспроизводит анимации PPTX на 100%. Стратегия APS — автоматизация нативного PowerPoint — является единственным проверенным и надёжным подходом.

2. **Рынок не предлагает готового open-source решения** с единым UI для управления библиотекой, предпросмотра и бесшовного переключения между PPTX/PDF/видео.

3. **Есть ниша для приложения с лучшим UX**, которое объединяет управление всеми типами контента в одном интерфейсе с предварительным просмотром и навигацией по слайдам.

## 3. Архитектурное решение

### 3.1. Критическое решение: показ PPTX

> **Рекомендация:** Не конвертировать PPTX. Управлять нативным PowerPoint / Keynote программно.

**Обоснование:**
- PowerPoint — единственный рендерер, который на 100% воспроизводит свои собственные анимации, переходы, встроенное видео, шрифты и макросы.
- Конвертация в HTML/PDF/изображения гарантированно теряет: Morph-переходы, пользовательские motion path, trigger-анимации, встроенные видео, 3D-модели.
- Профессиональный стандарт индустрии — именно автоматизация нативного приложения.

**Механизм:**
- **Windows:** COM Automation API для PowerPoint (через Node.js addon или вызов PowerShell/C# скрипта).
- **macOS:** AppleScript / JXA (JavaScript for Automation) для PowerPoint for Mac или Keynote.

### 3.2. Платформа приложения

**Рекомендация: Electron**

| Критерий | Electron | Tauri |
|----------|----------|-------|
| Экосистема | Зрелая, огромная | Растущая |
| Multi-window / Dual-screen | Полная поддержка | Есть, но с багами (масштабирование между мониторами) |
| Вызов нативных процессов | `child_process`, N-API addons | Rust + sidecar |
| Сообщество и примеры | Огромное | Меньше |
| Размер бандла | ~150 MB | ~5–10 MB |
| Потребление RAM | Выше | Ниже |

Electron выбран потому что:
- Нужна надёжная работа с двумя мониторами (окно управления + полноэкранный вывод).
- Нужна интеграция с COM/AppleScript — проще через `child_process` и готовые Node.js-модули.
- Размер бандла некритичен для десктопного приложения данного класса.

## 4. Функциональные требования

### 4.1. Библиотека презентаций

- Загрузка папки с файлами (PPTX, PDF, MP4/MOV/AVI).
- Древовидное или плиточное отображение содержимого папки.
- Фильтрация по типу (PPTX / PDF / Видео).
- Drag & drop для изменения порядка показа.
- Запоминание последней открытой папки.

### 4.2. Показ PPTX (с анимациями и видео)

- Запуск SlideShow нативного PowerPoint / Keynote на выбранном мониторе.
- Управление: следующий слайд, предыдущий, переход к конкретному номеру.
- «Заморозка» экрана (screenshot overlay) во время переключения между файлами.
- Поддержка всех типов анимаций, переходов и встроенного видео — без конвертации.

### 4.3. Показ PDF

- Рендеринг через pdf.js в окне презентации (Monitor 2).
- Полноэкранный режим, навигация по страницам.
- Предварительный просмотр миниатюр страниц в окне управления.

### 4.4. Воспроизведение видео

- HTML5 `<video>` для MP4/MOV/WebM в окне презентации.
- Альтернатива: интеграция с VLC (libVLC) для широкой поддержки кодеков.
- Управление: play, pause, stop, перемотка, громкость, прогресс-бар.
- Зацикливание (loop).

### 4.5. Предварительный просмотр

- PPTX: генерация миниатюр слайдов (через COM API или LibreOffice в headless-режиме для превью).
- PDF: рендеринг миниатюр через pdf.js.
- Видео: генерация стоп-кадра через ffmpeg.
- Панель превью слайдов с возможностью клика для перехода к конкретному слайду.

### 4.6. Навигация и управление

- Переход к слайду по номеру (текстовое поле с мгновенным переходом).
- Горячие клавиши: вперёд/назад, ESC для выхода, пробел для паузы видео.
- Поддержка кликера (презентер-пульт) — стандартные HID-клавиши Page Up/Down, стрелки.
- Индикатор текущего слайда / общего количества.

### 4.7. Бесшовное переключение

- При переключении между файлами — захват скриншота текущего состояния экрана и показ поверх всего, пока загружается следующий контент.
- Fade-out / fade-in переход (настраиваемое время).
- Возможность показа нейтрального экрана (чёрный, логотип) между презентациями.

## 5. Нефункциональные требования

- **Платформы:** Windows 10/11, macOS 12+.
- **Предусловия:** Для показа PPTX с анимациями на машине должен быть установлен Microsoft PowerPoint (Windows) или PowerPoint for Mac / Keynote (macOS).
- **Производительность:** Переключение между файлами < 3 секунд (с overlay).
- **Стиль:** Тёмная тема, минималистичный современный дизайн.
- **Мониторы:** Автоматическое определение подключённых мониторов, выбор целевого дисплея.

## 8. Сценарии использования (User Flows)

### Сценарий 1: Конференция с несколькими докладчиками

1. Оператор загружает папку с 10 файлами PPTX от разных спикеров.
2. В библиотеке отображаются все файлы с миниатюрами первых слайдов.
3. Оператор выбирает первую презентацию — PowerPoint запускает SlideShow на втором мониторе.
4. Спикер листает слайды кликером (Page Down → перехватывается приложением → передаётся PowerPoint).
5. Доклад окончен. Оператор нажимает «Следующая презентация» — экран на долю секунды показывает overlay (последний слайд или логотип), пока загружается следующий файл.
6. Процесс повторяется.

### Сценарий 2: Смешанный контент (PPTX + видео + PDF)

1. Папка содержит: intro.mp4, keynote.pptx, agenda.pdf, promo.mp4.
2. Оператор запускает intro.mp4 — видео играет на втором мониторе, в окне управления видны контролы воспроизведения.
3. Видео заканчивается → оператор переключается на keynote.pptx → бесшовный fade.
4. После доклада — быстрый показ agenda.pdf, затем promo.mp4.

## 9. Риски и ограничения

| Риск | Влияние | Митигация |
|------|---------|-----------|
| PowerPoint не установлен | PPTX показать нельзя | Проверка при запуске, уведомление пользователю. Fallback: показ слайдов как изображений (без анимаций). |
| COM API различается между версиями PowerPoint | Несовместимость | Тестирование на PowerPoint 2016, 2019, 2021, 365. |
| macOS ограничивает AppleScript для безопасности | Не работает автоматизация | Запрос разрешений Accessibility, инструкция пользователю. |
| Тяжёлые видео тормозят | Плохой UX | Аппаратное ускорение, предзагрузка. |
| Кликер работает глобально | Конфликт с другими приложениями | Перехват горячих клавиш только в фокусе. |

## 10. Этапы разработки (Roadmap)

### Фаза 1 — MVP (8–10 недель)

- Electron-приложение с двумя окнами (управление + презентация).
- Загрузка папки, отображение списка файлов.
- Показ PDF (pdf.js) на втором мониторе.
- Воспроизведение видео (HTML5) с базовыми контролами.
- Тёмная тема.

### Фаза 2 — PPTX-интеграция (6–8 недель)

- COM Automation для PowerPoint на Windows.
- AppleScript-мост для macOS.
- Управление слайдами из приложения (вперёд, назад, к номеру).
- Overlay для бесшовного переключения.
- Поддержка кликера.

### Фаза 3 — Полировка (4–6 недель)

- Генерация миниатюр для всех типов файлов.
- Панель предпросмотра слайдов.
- Drag & drop порядка файлов.
- Настройки (выбор монитора, время fade, горячие клавиши).
- Расширенное управление видео (перемотка, громкость, loop).

### Фаза 4 — Релиз (2–4 недели)

- Сборка и подпись для Windows (NSIS/MSI) и macOS (DMG).
- Авто-обновления.
- Документация пользователя.
- Бета-тестирование.

## 12. Заключение

Проект закрывает реальную нишу: на рынке отсутствует бесплатное кросс-платформенное приложение с единым UI для управления показом PPTX (с полным сохранением анимаций), PDF и видео на дополнительном экране. Ключевое архитектурное решение — не пытаться рендерить PPTX самостоятельно, а автоматизировать нативный PowerPoint — проверено индустрией и является единственным путём к 100% совместимости с анимациями и встроенным видео.
